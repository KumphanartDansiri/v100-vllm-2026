# Chapter 1 — V100 in 2026: vLLM 0.21 source build + modern model support

> **Status: DRAFT** — numbers provisional until the final freeze rerun ([docs/FINAL_RERUN.md](FINAL_RERUN.md)). Tables auto-render from `data/benchmark_matrix.csv`.


**Hook:** vLLM 0.21's pip wheels dropped practical V100 support, but the source still has an sm_70
path if built with CUDA 12.6. Here are 6 modern model families running on 8×V100-32GB in 2026.

## Why the wheels don't work but the source does
- 0.21 CMake: the CUDA ≥12.8 arch branch starts at `7.5` (no V100); the `<12.8` branch still lists
  `7.0`. Prebuilt wheels are ≥12.8 builds → no sm_70. **Build from source on CUDA 12.6 → sm_70 is
  back, no source patch.** (0.19 context: sm_70 was present more broadly; verify exact wheel status
  vs vLLM release notes before quoting "official".)
- Flags that matter on V100: `--dtype float16`, `cudagraph_mode=FULL_DECODE_ONLY`,
  `--skip-mm-profiling` (text-only; ~45× faster cold start), TRITON_ATTN auto-selected.
