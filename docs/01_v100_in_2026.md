# Chapter 1 — V100 in 2026: a vLLM 0.21 source build + 6 modern model families

> **Status: DRAFT** — numbers provisional until the final freeze ([FINAL_RERUN.md](FINAL_RERUN.md)). Tables auto-render from `data/benchmark_matrix.csv`.

Tesla V100s are everywhere on the surplus market and treated as e-waste by modern inference stacks.
They are not e-waste. With one build trick you can run current Qwen, Gemma, and GLM models — dense
and MoE, FP16 and FP8 — on 8×V100-32GB with vLLM 0.21 today.

## The build trick (and the honest version of "supported")

It would be wrong to say "vLLM 0.21 supports V100." The accurate statement:

> **vLLM 0.21's prebuilt pip wheels dropped practical V100 support** — they're built on CUDA ≥12.8,
> whose arch list starts at sm_75. **But vLLM's *source* still has an sm_70 path**: its CMake
> `<12.8` branch still lists `7.0`. So **building 0.21 from source on a CUDA 12.6 toolchain
> re-enables Volta with no source patch.**

That's the whole unlock. You don't get it from `pip install vllm`; you get it from a source build on
the 12.6 toolchain (image we use: `vllm-v100:vllm021-cu126`). vLLM 0.19 carried sm_70 more broadly,
but pinning 0.19 strands you on older model support — the point of this project is staying current.

## Flags that matter on V100
- `--dtype float16` (V100 has no bf16 tensor-core path worth using; fp16 is home turf).
- `--compilation-config '{"mode":0,"cudagraph_mode":"FULL_DECODE_ONLY"}'` — cudagraph is the real
  serving baseline (Chapter 3); it works on Volta and is a 3–5× lift over eager.
- `--skip-mm-profiling` for text-only serving — multimodal models (Gemma, Qwen-VL) otherwise run a
  giant dummy-image profile at startup; skipping it cuts cold start ~45× with zero decode impact.
- TRITON_ATTN is auto-selected on sm_70.

Reading the table: FP16-MoE rows are labelled `stock(pre-moe-patch)` — those are the *un-fixed*
numbers; Chapter 2 is the 4–9× fix. FP8 rows (`fp8-plugin+coalesced`) use our custom sm_70 W8A16
plugin (Chapter 5) — after a branchless-dequant rewrite it is now a decode-*speed* path for
low-user and large-MoE decode, not only a residency trick. Dense fp16 / GPTQ-int4 are `stock-vllm`. All loaded and generated coherent text
on V100 — "old but not dead" is a measured claim, not a slogan.
